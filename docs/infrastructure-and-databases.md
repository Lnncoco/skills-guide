# 数据库与基础设施

这一页集中放数据库、存储、容器化和运维相关的技能。

---

## 数据库与存储

- **`sql-best-practices`** ([sanjeed5/awesome-cursor-rules-mdc](https://github.com/sanjeed5/awesome-cursor-rules-mdc))
    - **介绍**：通用 SQL 安全与规范指南。
    - **场景**：你不是只想写出能跑的 SQL，而是想先把命名、注入风险和结构规范定住时。

- **`postgres-best-practices`** ([neondatabase/postgres-skills](https://github.com/neondatabase/postgres-skills))
    - **介绍**：PostgreSQL 架构与查询优化指南。
    - **场景**：数据库明确是 PostgreSQL，且你开始关心查询优化、索引和架构习惯时。

- **`mysql-best-practices`** ([planetscale/database-skills](https://github.com/planetscale/database-skills))
    - **介绍**：面向大规模 MySQL 的性能架构指南。
    - **场景**：项目使用 MySQL，且你要处理分页、事务一致性或大规模场景下的性能约束时。

- **`sqlite-best-practices`** ([tursodatabase/agent-skills](https://github.com/tursodatabase/agent-skills))
    - **介绍**：SQLite 嵌入式 / 边缘数据库性能指南。
    - **场景**：你用的是 SQLite，不想把服务端数据库的习惯原样搬过来时。

## 基础设施与运维

- **`bash-scripting`** ([mindrally/skills](https://github.com/mindrally/skills))
    - **介绍**：覆盖 Bash 脚本的常见工程写法和自动化习惯，适合处理开发中高频但不追求深运维的命令行任务。
    - **场景**：你要写部署辅助脚本、环境初始化脚本、简单自动化流程，或者整理日常 Linux 命令用法时。

- **`docker-containerization`** ([ailabs-393/ai-labs-claude-skills](https://github.com/ailabs-393/ai-labs-claude-skills))
    - **介绍**：围绕 Dockerfile、Compose、镜像组织和基础容器化流程的常见做法，适合处理中小型项目的容器落地。
    - **场景**：要做简单容器化、本地到测试环境的部署收口，或者整理镜像和运行方式时。
