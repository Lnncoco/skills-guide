# 元技能与辅助

这一页集中放技能发现、技能编写、文档措辞重写和沟通模式这类辅助能力。

---

- **`find-skills`** ([vercel-labs/skills](https://github.com/vercel-labs/skills))
    - **介绍**：在技能生态中搜索并评估新技能。
    - **场景**：现有 skill 不够用，或者你怀疑某个细分能力已经有现成 skill 可装时。

### 文档措辞、重写与风格

- **`technical-writing`** ([supercent-io/skills-template](https://github.com/supercent-io/skills-template))
    - **介绍**：聚焦技术文档的结构、表达和读者导向，也能处理措辞、句子密度、术语落点和说明清晰度。
    - **场景**：你手里已经有技术内容，但需要把它整理成更像正式开发文档的表达时。

- **`writing-clearly-and-concisely`** ([softaworks/agent-toolkit](https://github.com/softaworks/agent-toolkit))
    - **介绍**：用更明确、更克制的写法重写说明文字，重点处理空话、赘词、被动语态和 AI 味过重的句子。
    - **场景**：你不是缺结构，而是已经能明确指出“这段太虚、太绕、太像套话”时。

- **`documentation-writer`** ([github/awesome-copilot](https://github.com/github/awesome-copilot))
    - **介绍**：偏通用文档作者视角，擅长把零散内容整理成结构完整、读者导向更强的说明文档。
    - **场景**：手里已经有笔记、草稿、需求对话或分散段落，需要重写成正式文档时。

- **`gws-docs-write`** ([googleworkspace/cli](https://github.com/googleworkspace/cli))
    - **介绍**：偏正式文档写作流程，适合把散乱素材整理成更成型、更稳定的说明文本。
    - **场景**：文档已经不只是几段说明，而是开始需要稳定章节、统一语气和可反复维护的写作方式时。

- **`tech-doc-style-chinese`** ([aradotso/trending-skills](https://github.com/aradotso/trending-skills))
    - **介绍**：面向中文技术文档的表达风格修订，强调中文语序、术语一致性和说明节奏。
    - **场景**：主要写中文文档，且你觉得问题不是“没内容”，而是“中文技术表达不自然、不利落、不统一”时。
    - **补充**：更像风格专项补充，适合和通用的 `technical-writing` 配合使用。

### 📦 [mattpocock/skills](https://github.com/mattpocock/skills)

- **`write-a-skill`** ([mattpocock/skills](https://github.com/mattpocock/skills))
    - **介绍**：辅助设计并编写符合标准的自定义 skill，包括 `SKILL.md` 和目录结构。
    - **场景**：某套做事方法已经重复出现，值得沉淀成自己可复用的 skill 时。

- **`caveman`** ([mattpocock/skills](https://github.com/mattpocock/skills))
    - **介绍**：切换至极致压缩的低 Token 沟通模式。
    - **场景**：你明确希望输出更短、更少铺垫、更低 token 消耗时。
